# 用户阶段文档模板

为同一学习主线维护一份滚动 `stage-guide.md`。该文档面向用户；只保留可复用的实验核心，不复制 AI 状态机、内部交接字段或对话流水账。

阶段开始时写实验入口，结果区保持“待实验”；阶段结束时用真实证据完成同一节，并追加下一阶段入口。按主题删减不适用字段。

```markdown
# <实验主题>阶段指南

> 证据等级：概念推断 / 仿真证据 / 台架实测 / 真实工况验收

- 适用假设：
```

## 更新约束

- 用表格保存多方案指标或重复字段；其余内容优先使用短段落和短列表。
- 阶段结束后压缩过程，只保留实验问题、关键配置、真实结果、因果结论、工程动作、边界和缺口。
- “待实验”只能由真实用户反馈或工具输出替换；预测与结果分开记录。
- 带缺口结束时明确用户决定、缺口和对后续结论的影响，不写成完整通过。
- 下一阶段尚未确认时保持“未切换”；确认后才追加下一阶段实验入口。
