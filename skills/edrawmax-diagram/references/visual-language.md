# 图表视觉语言

## 先选表达范式

| 信息关系 | 优先范式 | 重点 |
|---|---|---|
| 有明确执行顺序 | 流程图 / 控制框图 | 主链、分支、反馈 |
| 模块分层与依赖 | 架构图 / 分层框图 | 边界、职责、接口 |
| 时间先后与里程碑 | 时间线 / 阶段图 | 时间刻度、阶段变化 |
| 多项并列或递进 | 列表图 / 步骤图 / Chevron | 顺序、层次、完成度 |
| 中心主题向外发散 | 思维导图 / 概念图 | 分类、关联、层级 |
| 集合、占比或循环 | 维恩图 / 环形图 / 饼图 | 数量关系或重叠 |
| 专业设备与拓扑 | 网络图 / 电气图 / 平面图 | 标准符号、端口、连接 |

不要把上述范式统一画成圆角矩形串联。

## 形状语义

| 语义 | 图形 |
|---|---|
| 开始、结束、稳定状态 | 椭圆或胶囊形 |
| 过程、模块、任务 | 矩形或圆角矩形 |
| 条件、判断、分流 | 菱形 |
| 输入、输出、外部命令 | 平行四边形 |
| 队列、缓存、持久化数据 | 圆柱体 |
| 文档、报告、清单 | 文档形 |
| 子流程、可展开模块 | 双边框子流程 |
| 阶段推进 | Chevron 箭头块 |
| 注释、限制、注意事项 | 便笺或标注符号 |
| 设备、传感器、网络、电气元件 | 对应专业原生符号 |

## 视觉通道分工

- 形状：对象种类。
- 颜色：职责域、状态或风险级别；同一职责域保持同色系。
- 实线：主流程、数据流或强依赖。
- 虚线：控制、反馈、异步通知或可选关系。
- 点线：关联、注释、监测或弱依赖。
- 线宽：重要程度；主链略粗，辅助线较细。
- 分组容器：模块边界、线程域、硬件/软件域或泳道。

每个视觉通道尽量只表达一个维度，避免“红色既表示错误，又表示硬件，还表示关键链路”。

## 层级与密度

- 总览图：5～9 个一级节点，每个节点 1 个标题加最多 2 行短说明。
- 详细图：只展开一个一级节点，允许状态、参数和异常分支。
- 重要对象尺寸更大或位于视觉中心；辅助任务放在侧面或底部。
- 主链间距一致，反馈线留出独立通道，不贴边绕行。
- 图题说明“这张图回答什么问题”，副标题可说明主循环、中断、硬件等职责分工。

## 参考图提取清单

看到参考图后依次记录：

1. 画布方向与主链方向。
2. 一级节点数量、位置与宽高比例。
3. 是否有上下分层、泳道、容器或外围反馈。
4. 形状分别表达什么语义。
5. 实线、虚线、颜色和箭头分别表达什么关系。
6. 视觉重心和最重要的 1～2 条链路。

复刻时优先保持拓扑、层级、反馈路径和形状语法；文字可压缩，关键关系不可省略。
